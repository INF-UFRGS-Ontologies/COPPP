Axiomatization Decision LogRegistro de Decisões
1 Identity-Giving FunctionFunção Identitária
Function class
FlowControlFunction ⊑ DesignedFunction
Realized in
FlowControlProcess ⊑ PlannedProcess
The identity-giving function is the ability to control fluid flow - to permit, obstruct, regulate or divert a fluid stream. This function is what makes a valve a valve regardless of its physical mechanism. Reusing the existing FlowControlFunction from already-enriched classes.
2 Genus DeterminationDeterminação do Gênero
MaterialArtifactbfo:Object (BFO_0000030)
While 'FLUID REGULATOR' appears in ISO 15926, this is too domain-specific and not yet enriched. Valve is a foundational equipment category in industrial ontologies. Each valve instance is a single causally unified object with a designed function, making it a MaterialArtifact. The parent candidates show MaterialArtifact as appropriate, and it is already enriched.
3 Differentia SpecificaDiferença Específica
∃bearerOf.FlowControlFunction

An artifact that bears a flow control function to permit, obstruct, regulate, or divert fluid streams.

The flow control function is the minimal differentiating condition. MaterialArtifact already provides the designed function constraint, so we specify the particular type of function that makes something a valve.

TransformationTransformação
flow control without energy addition - permits, obstructs, regulates, or diverts fluid
Valves are single-input, single-output devices that modify flow characteristics without adding energy to the fluid. The fluid composition remains unchanged, only flow parameters are controlled.
Disjoint WithDisjunto com

These equipment types have incompatible process signatures - pumps and compressors add energy to fluids, turbines extract energy from fluids, while valves control flow without energy transfer.
